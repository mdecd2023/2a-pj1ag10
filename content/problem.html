problem

CoppeliaSim Edu use language

CoppeliaSim Edu 使用 6 種不同編碼語言的強大 API，在這6種編碼語言中 C++ 和 Python 是模擬器中使用最多的語言，並且還必須了解 Python 及 Lua language。

相關參考資料：https://robocraze.com/blogs/post/understanding-simulators-through-coppliasim

Lua language

Lua 是在1993年由羅伯托·耶魯薩林斯希、Luiz Henrique de Figueiredo 和 Waldemar Celes 建立的，他們當時是巴西的里約熱內盧天主教大學的電腦圖形技術組（Tecgraf）成員，而 Lua 的先驅是資料描述/組態語言「SOL」（簡單物件語言）和「DEL」（資料錄入語言）

相關參考資料：https://zh.wikipedia.org/zh-tw/Lua

ttt 檔與 xml 檔

ttt 檔通常是由 Interscape 開發稱為TestTrack Pro的軟體，它們可以歸類為三種主文件類型。 通常，這些文件被視為TestTrack Pro Database File格式。 在大多數情況下，這些文件被視為Data Files, 但它們也可以是Encrypted Files。

相關參考資料：https://www.solvusoft.com/zh-tw/file-extensions/file-extension-ttt/

xml 檔為可延伸標記式語言（英語：Extensible Markup Language，簡稱：XML）是一種標記式語言。XML是從標準通用標記式語言（SGML）中簡化修改出來的。它主要用到的有可延伸標記式語言、可延伸樣式語言（XSL）、XBRL和XPath等。

xml 設計是用來傳送和攜帶資料資訊，不用於表現和展示資料，HTML則用來表現資料，所以XML用途的焦點是在於說明資料是什麼以及攜帶資料資訊。

相關參考資料：https://zh.wikipedia.org/zh-tw/XML

coppeliasim 中的 scene 與 model 有何不同?

scene：與模型相比，場景可以包含完全相同類型的元素，但還包括特定於場景的環境、主腳本、頁面和檢視、多個相機對象、幾個發光的物體、多個視圖、多個頁面、地板及默認主腳本。

model：模型是場景的子元素。模型本身不能存在，除非存在於文件（“*.ttm”-文件類型）中，也不能通過自身進行模擬。模型必須包含在場景中才能運行。
